2. Product Overview

The GPF-2X72C software controls our bus master digital input/output products from your application running on Windows. Application software should link a provided dynamic link library (DLL) and control the bus master digital input/output products  through the application programming interface (API).

Q14) Triggers such as start trigger by the START signal and end trigger by the END signal are not generated in the second and later starts.
A14) For bus master digital input and bus master digital output products, the trigger conditions including a start trigger and end trigger are enabled only in the first sampling start after opening the device. The conditions are not applied for the second or later start by the DioBmStart function.
Or, consider the settings of external interfaces to use the gated-sampling mode and gated-pattern mode.
(The gated-sampling mode and gated-pattern mode allow the device to input and output data only while the GATE signal is asserted.)

Q15) When the buffer I specify greater than or equal to 64 MB, the DioBmStart function returns DIOBM_ERROR_ANOTHER when opening a board or card. 
A15) The bus master transfer buffer size must be specified less than 64 MB. The system call cannot simultaneously handle the area of greater than 64 MB. If the buffer size is specified greater than or equal to 64 MB, the DioBmStart function returns DIOBM_ERROR_ANOTHER when opening a board or card. 

If the input/output area is used for the buffer, a large amount of data can be sent/received. When the amount of input/output data is greater than the buffer size, the data is returned to the head of the buffer and input/output data is continued after transmitting the last data. For data output, you can output seamlessly large amount of data due to making update the transferred data area periodically. For data input, you can input consecutively large amount of data by moving the sampled data to another buffer.
 
Q16) How fast the data transfer rate when I use two or more bus master digital input/output devices. 
A16) When you use multiple devices at the same time, each board cannot operate at its maximum transfer rate .

The  total amount of all data transfer of all devices used at the same time is about 80 MB/s at a maximum. 

When you use  high-load device with PCI bus, consider the load of the device. 

The transfer rate of PCI bus is 132 MB/s with bus clock 33MHz, bus width 32 bits(4 bytes). 
Considering the clock consumption other than data transfer, the maximum amount of data transfer is about 80 MB/s.   

Therefore, the maximum clock setting value is 20 MHz in our bus master digital input/output products（20 MHz * 32 bit（4 bytes）= 80 MB/s）.

The maximum clock setting value is set assuming only one board is used. If you use multiple boards, consider the total amount of data transfer at the same time to be within about 80 MB/s. With the consideration, data missing can be avoided in the mode that does not perform data synchronization.
 
Q17) What is the relationship between the buffer size specified by the DioBmSetBufferConfig function and the number of items specified by the DioBmSetDeviceConfig and DioBmSetDeviceConfigEx functions?
A17) The DioBmSetBufferConfig function configures only the transfer destination buffer. 

It does not affect the number of input/output specified by the DioBmSetDeviceConfig and DioBmSetDeviceConfigEx functions.

In data input, if lager data than configured buffer are input, the program overwrites from the initial data. If the data are smaller than the configured buffer, unused buffer will be exist.  

In data output, if the buffer size is smaller than the specified number of output data, the program outputs data to the last and returns the first to continue output. The program can continuously output data with a minimum buffer to output the target data. Configure the buffer size in accordance with the purpose.

When buffer size (in byte) and the number of data output are set to the same value, the program outputs all data in the buffer.  
 
Q18) When the transfer width is WORD, what is the unit of  input/output counter and DMA transfer counter?
A18) The retrieved number is the number of items.

To find the byte count, double the number of items.

The input/output counter counts up one by one, while the DMA transfer counter counts up two by two (double word).
 
Q19) What is difference between DIOBM_EV_ENDTRG and DIOBM_EV_DMAEND events?
A19) These two events differs in the generation timing. That is because the FIFO memory is mounted on the device. 
The DIOBM_EV_ENDTRG event  is generated when the specified number of input or output is completed.  In the case of input, data may be left in FIFO, and in the case of output, all data are output. 
The DIOBM_EV_ENDTRG event  is generated when input is completed, and all data including data  in FIFO are DMA transferred to the system. In the case of output, the event is generated when all output data are DMA transferred to the device. Data may be left in FIFO. 
 
If you want to receive events in the status that data can be retrieved, use DIOBM_EV_DMAEND.

If completion of DMA transfer is not required, use DIOBM_EV_ENDTRG.
(For events corresponding to the number of completion of  input: such as monitoring and displaying the number of input.) 
 
Q20) Abnormal termination occurs when I input data. The intended data cannot be output. 
A20) Check that the transfer buffer has the possibility to be released or deleted during transfer. 
If the transfer buffer is a local variable, consider the variable scope.

In the bus master transfer, data are directly transferred to the user mode address specified by the DioBmSetBufferConfig function.
Therefore, unexpected memory region may be rewritten or unintended data may be output if the buffer is released or deleted during transfer.  
 
Q21) If the program is forcibly terminated during data transfer, what happens to the data left in FIFO of the device?   
A21) If the program is terminated by the DioBmEnd function or external signal, the data left in FIFO of the device are cleared at the start of next data input/output.  

If the program is closed by the DioBmClose function and opened again, the data left in FIFO of the device are cleared at the start of next data input/output.  

If the program is suspended by the DioBmStop function or STOP signal, the data left in FIFO of the device are not cleared when the program resumes. 
 
Q22) In the PCI-2172C, PCI-2472C, PCI-2772C, CPZ-291244, CPZ-292244, CBI-291144, and CBI-292144, the memory is described as 508x32bit FIFO, 511x32bit FIFO, or 2045x32bit FIFO. Can I transfer larger data than the FIFO memory at a time. 
A22) FIFO memory is used as an intermediate buffer. It absorbs the velocity difference between PCI bus and sampling or output circuit. It avoid interruption of output and missing of input data. 
 
Q23) Can I use the DioBmSetDeviceConfigEx, DioBmGetDeviceConfigEx, DioBmStopEx, and  DioBmSoftTriggerEx functions for the PCI-2772C, CPZ-291244, CPZ-292244, CBI-291144, and CBI-292144 in the PCI-2172C and PCI-2472C?
A23) These functions for the PCI-2772C, CPZ-291244, CPZ-292244, CBI-291144, and CBI-292144 can be used in the PCI-2172C and PCI-2472C. 
However, the parameters added for the PCI-2772C cannot be specified in the PCI-2172C and PCI-2472C.
 
Q24) The following functions can be used without preparing the data transfer buffer in application.  
DioBmAllocateBuffer, DioBmFreeBuffer, DioBmGetData, DioBmReadData, DioBmSetData, DioBmWriteData, DioBmClearData

When should I use these functions?
A24) These functions that do not require the data transfer buffer in application can be used without creating wrapper component separately when it is difficult to allocate memory dynamically on  the graphical development environment such as LabVIEW.  

By simplifying the code to write data and to control the pointer needed for retrieving data, the programming becomes easy.
